If you believe you have discovered a security vulnerability in any NVIDIA product — including the Open GPU Kernel Modules — please report it through one of the secure channels below. Do not open a public GitHub issue.

## Reporting channels

Use either of the following methods to report a potential vulnerability:

<CardGroup cols={2}>
  <Card title="Web form" icon="globe">
    Submit via the [Security Vulnerability Submission Form](https://www.nvidia.com/object/submit-security-vulnerability.html) on the NVIDIA website.
  </Card>
  <Card title="Email" icon="envelope">
    Send an encrypted email to [psirt@nvidia.com](mailto:psirt@nvidia.com). Encrypt your message using NVIDIA's PGP key before sending.
  </Card>
</CardGroup>

<Note>
  **OEM Partners:** Contact your NVIDIA Customer Program Manager directly rather than using the channels above.
</Note>

<Warning>
  Do **not** report security vulnerabilities as public GitHub issues. Public disclosure before a fix is available puts users at risk. Use the secure reporting channels described on this page.
</Warning>

## What to include in an email report

When reporting via email, encrypt your message with [NVIDIA's PGP key](https://www.nvidia.com/en-us/security/pgp-key/) and include the following information:

- **Product and version:** The product or driver name, and the version or branch that contains the vulnerability.
- **Vulnerability type:** The category of the issue (for example: code execution, denial of service, buffer overflow, privilege escalation).
- **Reproduction steps:** Clear instructions to reproduce the vulnerability.
- **Proof-of-concept or exploit code:** Any code or commands that demonstrate the issue.
- **Potential impact:** A description of the impact and how an attacker could exploit the vulnerability.
