---
title: "Building from Source"
description: "Build the NVIDIA open GPU kernel modules (version 595.58.03) from source on Linux."
---

This guide covers building version **595.58.03** of the NVIDIA open GPU kernel modules from source.

## System Requirements

Before building, ensure the following are installed on your system:

| Requirement | Details |
|---|---|
| **Linux kernel headers** | Must match the running kernel version |
| **GCC or Clang** | Any reasonably modern version |
| **make** | GNU make 3.80 or later |
| **binutils** | Provides `ld`, `ar`, `objcopy`, and related tools |
| **Linux kernel version** | 4.15 or newer |

<Note>
The kernel modules built from this source must be used with GSP firmware and user-space NVIDIA GPU driver components from the **matching 595.58.03 driver release**. Mismatched versions will not work correctly.
</Note>

## Build Steps

<Steps>
  <Step title="Install build dependencies">
    On Debian/Ubuntu-based systems:

    ```bash
    sudo apt-get install build-essential linux-headers-$(uname -r)
    ```

    On RHEL/Fedora-based systems:

    ```bash
    sudo dnf install kernel-devel kernel-headers gcc make
    ```
  </Step>

  <Step title="Build the kernel modules">
    From the root of the source tree, run:

    ```bash
    make modules -j$(nproc)
    ```

    This compiles the OS-agnostic components under `src/` and then invokes the kernel build system (Kbuild) for the kernel interface layers under `kernel-open/`.
  </Step>

  <Step title="Install matching user-space components">
    Before installing the kernel modules, install the user-space driver components from the corresponding `.run` package using the `--no-kernel-modules` flag to skip the bundled kernel modules:

    ```bash
    sh ./NVIDIA-Linux-[...].run --no-kernel-modules
    ```

    Replace `[...]` with the appropriate architecture and version string for your system.
  </Step>

  <Step title="Install the kernel modules">
    Uninstall any existing NVIDIA kernel modules first, then run as root:

    ```bash
    sudo make modules_install -j$(nproc)
    ```

    This installs the built `.ko` files into the kernel module tree using Kbuild's `modules_install` target.
  </Step>
</Steps>

## Output Location

After a successful build, the kernel module (`.ko`) files are placed in the `kernel-open/` directory alongside the kernel interface sources. The modules built are:

| Module file | Description |
|---|---|
| `kernel-open/nvidia.ko` | Core NVIDIA GPU driver module |
| `kernel-open/nvidia-drm.ko` | DRM/KMS integration module |
| `kernel-open/nvidia-modeset.ko` | Mode-setting module |
| `kernel-open/nvidia-uvm.ko` | Unified Memory driver module |

## Build Variables

The following variables can be passed on the `make` command line to control the build. For example:

```bash
make modules -j$(nproc) NV_VERBOSE=1
```

| Variable | Description |
|---|---|
| `NV_VERBOSE` | Set to `"1"` to print each complete command as it executes. By default, a short label (e.g., `CC`) is shown instead. |
| `DEBUG` | Set to `"1"` to compile with `-O0 -g` and enable additional debug log messages in the kernel modules. Release builds compile with `-O2`. |
| `TARGET_ARCH` | Target CPU architecture: `x86_64` or `aarch64`. Defaults to the host architecture. Required when cross-compiling. |
| `CC` | C compiler override (default: `gcc`). |
| `LD` | Linker override (default: `ld`). |
| `AR` | Archiver override (default: `ar`). |
| `CXX` | C++ compiler override (default: `g++`). |
| `OBJCOPY` | Object copy tool override (default: `objcopy`). |

<Note>
The kernel interface layer of each module must be built with the same toolchain that was used to build the target kernel. The build system will attempt to detect the correct compiler automatically from the kernel's `CONFIG_CC_VERSION_TEXT` configuration value.
</Note>
