---

This guide walks you through cloning the repository, installing build prerequisites, compiling the kernel modules, and installing them alongside matching user-space driver components.

<Warning>
  The open kernel modules require **GSP firmware and user-space components from a matching driver release** (595.58.03). Install the user-space components before loading the open kernel modules, or the driver will not initialize correctly.
</Warning>

## Prerequisites

Before you begin, make sure you have the following:

| Requirement | Details |
|---|---|
| Linux kernel | 4.15 or newer, with kernel headers installed |
| Compiler | GCC or Clang (any reasonably modern version) |
| Build tools | `make`, `ld`, `ar` |
| NVIDIA user-space driver | Version 595.58.03 `.run` package, installed with `--no-kernel-modules` |
| GPU | Turing architecture or later (RTX 2000 series and newer) |

<Note>
  The kernel interface layer must be compiled with the same toolchain used to build your running kernel. On most distributions this means using the distribution-packaged GCC.
</Note>

## Step-by-step installation

<Steps>
  <Step title="Clone the repository">
    Clone the open GPU kernel modules source from GitHub:

    ```bash
    git clone https://github.com/NVIDIA/open-gpu-kernel-modules.git
    cd open-gpu-kernel-modules
    ```

    The repository contains both the kernel interface layer (`kernel-open/`) and the OS-agnostic source (`src/`). See the [build from source guide](/building/build-from-source) for a detailed explanation of the directory layout.
  </Step>

  <Step title="Install build prerequisites">
    Install the kernel headers and build tools for your distribution:

    <Tabs>
      <Tab title="Ubuntu / Debian">
        ```bash
        sudo apt-get update
        sudo apt-get install -y \
          linux-headers-$(uname -r) \
          build-essential \
          gcc \
          make
        ```
      </Tab>
      <Tab title="RHEL / Fedora">
        ```bash
        sudo dnf install -y \
          kernel-devel-$(uname -r) \
          kernel-headers-$(uname -r) \
          gcc \
          make
        ```

        On RHEL/CentOS you may need to enable the `BaseOS` and `AppStream` repositories, or install the `-devel` package matching your exact running kernel version.
      </Tab>
      <Tab title="Arch">
        ```bash
        sudo pacman -Sy --needed \
          linux-headers \
          base-devel
        ```

        If you are running a non-standard kernel (e.g., `linux-lts`), install the corresponding `linux-lts-headers` package instead.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Build the kernel modules">
    Run `make` from the repository root to build all four kernel modules:

    ```bash
    make modules -j$(nproc)
    ```

    The build compiles the kernel interface layer for your currently running kernel and links it with the pre-built OS-agnostic component. The resulting `.ko` files are placed under `kernel-open/`.

    See the [build variables reference](#build-variables) below for options to control verbosity and debug output.
  </Step>

  <Step title="Install user-space components">
    Before installing the kernel modules, install the matching NVIDIA user-space driver components. Use the `--no-kernel-modules` flag to skip the proprietary kernel modules included in the `.run` package:

    ```bash
    sh ./NVIDIA-Linux-x86_64-595.58.03.run --no-kernel-modules
    ```

    Replace the filename with the `.run` package for your architecture (`x86_64` or `aarch64`) and the 595.58.03 release. Download the package from the NVIDIA driver download page.

    <Warning>
      You must uninstall any existing NVIDIA kernel modules before installing the open modules. Verify that old modules are removed first:

      ```bash
      sudo rmmod nvidia_uvm nvidia_drm nvidia_modeset nvidia
      ```
    </Warning>
  </Step>

  <Step title="Install the kernel modules">
    Install the compiled kernel modules as root:

    ```bash
    sudo make modules_install -j$(nproc)
    ```

    This copies the `.ko` files into the appropriate location under `/lib/modules/$(uname -r)/` and runs `depmod` to update the module dependency map.

    After installation, load the modules and verify they are running:

    ```bash
    sudo modprobe nvidia
    nvidia-smi
    ```
  </Step>
</Steps>

<Tip>
  To have the kernel modules automatically rebuilt whenever you update your kernel, set up DKMS (Dynamic Kernel Module Support). DKMS registers the module source and rebuilds it on each kernel upgrade without manual intervention. See the [DKMS guide](/building/dkms) for setup instructions.
</Tip>

## Build variables

You can customize the build by passing variables on the `make` command line.

<AccordionGroup>
  <Accordion title="NV_VERBOSE — control build output verbosity">
    By default, the build prints a short `CC` summary line for each compiled file. Set `NV_VERBOSE=1` to print the full compiler invocation instead:

    ```bash
    make modules -j$(nproc) NV_VERBOSE=1
    ```

    This is useful when diagnosing build failures or confirming which flags are being passed to the compiler.
  </Accordion>
  <Accordion title="DEBUG — enable debug builds">
    By default, the modules are built without debugging information. Set `DEBUG=1` to include debug symbols and enable additional kernel log messages from the driver:

    ```bash
    make modules -j$(nproc) DEBUG=1
    ```

    Debug builds are larger and may have slightly different performance characteristics. Use them for development and troubleshooting, not for production deployments.
  </Accordion>
</AccordionGroup>

Both variables can be combined:

```bash
make modules -j$(nproc) NV_VERBOSE=1 DEBUG=1
```
